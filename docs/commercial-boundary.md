# Commercial Boundary

CAVA is open enough to give the ecosystem a shared action-governance language,
but it is not a copy of OSuite Studio.

## Open in this repository

- Canonical action schema.
- CAVA-IR schema and validation.
- Deterministic hashing.
- Receipt construction and verification.
- Approval binding verification.
- Runtime adapter and parser-pack contracts.
- Disclosure-safe reference parser packs.
- OpenTelemetry-style and in-toto-style projection helpers.
- Local examples and tests.

## Commercial in OSuite

- Maintained production parser packs for real enterprise runtimes.
- Private parser-pack update channels and customer-specific enrichment.
- PCAA policy routing and final-authority workflows.
- Decision Score calibration and customer posture management.
- Bounded Action Firewall and Action Gate Leases.
- Agent Runtime Exposure Graph and incident reconstruction.
- Managed evidence replay, approval queues, exports, and assurance packets.
- Enterprise signer orchestration, private KMS/HSM, verifiable credentials, and
  optional external verifier or ledger integrations.

## Positioning

CAVA should be reusable without OSuite. OSuite should be valuable because it
runs CAVA inside a managed governance control plane with policy, approvals,
runtime coverage, evidence, support, and operational accountability.
